Why use Pies?

Human communication flows naturally. You don’t need to ask your friend about his “ear’s OS” before delivering an information. When people talk, things are quite simple and fun.

But the same is not true to humans’ devices. To see a full example, please refer to “Why Joe, Amanda and Mike are my friends, but their devices are not”.

What is a Pie

Pie is an acronym of Piece of information. Things we want to share daily are just pieces of info: parties invitations, places to go, recipes, reviews, quotes, business cards...

Today, those digital pieces of info are so stuck and tied to systems environments that we are constantly forgetting how easy and fun would be simply sharing them in a human-friendly way.

(Copying and pasting everything to e-mail isn't an effective solution, it’s a limited workaround)

Introduction to OpenPie

OpenPie adds a layer of freedom and fun to devices.

OpenPie is a simple combination of structured data and vocabulary, a convention about using existing technologies and concepts to make mobile applications talk to each other, regardless the systems constraints.

It’s also an open initiative.

Recipes

The basic concept behind OpenPie is outputing and inputing pies (pieces of information) through recipes.

Recipe = structured data (using schema.org vocabulary)

To make sense, any input/output application must be talking the same language, or, in a better definition, the same vocabulary.

The chosen vocabulary for OpenPie is schema.org. So, in addition, OpenPie is friendly to search engines and towards semantic web efforts.

Giving and receiving

JSON is the initial approach for transmitting pies because it is familiar to most programmers and completely language independent.

Code example

We want to create a pie for Joe’s party. This piece of info may contains:

  • Title: Joe’s Party
  • Description: Let’s rock and bake some pies!
  • Date: November/24/2012
  • Time: 14:50

A party is an event.

Schema.org defines a set of possible properties to describe it. That schema is available at http://schema.org/Event

The resulting pie is a JSON string containing all the information put into suitable properties:

{"itemtype":"http://schema.org/Event","name":"Joe's party","description":"Let's rock and bake some pies!","startDate":"2012-11-24T14:50"}

Note that itemtype refers to the chosen schema, so it is possible to the other side (the application that is receiving the JSON string) to understand what it is about and manage it the right way.
